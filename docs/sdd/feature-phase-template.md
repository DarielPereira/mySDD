# Feature Phase Checklist Template

Use this template when entering Stage IV to add a new feature to an existing project. Save a copy as `phase-<n>-feature-tasks.md` inside `docs/sdd/project-decisions/` (for example `phase-3-feature-tasks.md`).

## Feature: <feature-name>

### Entry conditions

- Author requested feature development or
- Stages I–III for the project are complete and the project is stable

### Purpose

Short statement of the feature goal and the user-facing value.

### Scope & Constraints

- What is included
- What is explicitly out of scope
- Any compatibility or API constraints

### Tasks (checklist)

- [ ] Task 1 — short description (owner: @name) — validation: how to confirm
- [ ] Task 2 — short description (owner: @name) — validation: how to confirm
- [ ] Task 3 — short description (owner: @name) — validation: how to confirm

### Validation

List unit/integration tests, acceptance criteria, and manual checks required to accept the feature.

### Rollback / Migration Notes

Describe how to revert or migrate if the feature introduces data or compatibility changes.

### Documentation

Which docs, READMEs, or changelogs to update.

### Release notes

Short bullet points suitable for a changelog.
