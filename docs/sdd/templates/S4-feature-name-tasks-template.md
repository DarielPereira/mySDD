# S4 — Feature Tasks (Template)

Use this template when introducing a new feature during Stage IV. Create one file per feature and
save the completed document in `docs/sdd/project-decisions/` as `S4-feature-<name>-tasks.md` (for
example, `S4-user-import-tasks.md`).

Stage IV feature work is tracked in a structure similar to Stage III phase work: the feature file
should be detailed enough that the implementation, validation, and follow-up work are clear from the
document alone.

- Related S2 phase plan:
- Related stack / data model notes:

## Purpose

Short statement of what the feature delivers and why it exists.

## Scope

Describe the in-scope behavior, expected user-facing or system-facing outcome, and the boundaries of
the feature.

## Tasks

List every task needed to complete the feature. Keep tasks small, concrete, and testable.

- [ ] Task 1 — action verb + concrete outcome — owner: @name — validation: exact check or command
- [ ] Task 2 — action verb + concrete outcome — owner: @name — validation: exact check or command
- [ ] Task 3 — action verb + concrete outcome — owner: @name — validation: exact check or command

For each task, include:

- What will change
- What will not change
- The owner (if known)
- The verification method
- Any dependencies on earlier tasks

## Validation

Describe the tests, checks, or manual verification steps required to accept the feature as complete.
Include command examples, expected outputs, or acceptance criteria where possible.

## Risks and follow-ups

List any risks, tradeoffs, deferred work, or follow-up tasks that should remain visible while the
feature is being executed.

- Design notes, diagrams, links to relevant code, tickets, or documents.
