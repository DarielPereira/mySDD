# S3 — Phase Tasks (Template)

Use this template to plan and track the tasks for one implementation phase. Create one completed
file per phase during Stage III and save it as `docs/sdd/project-decisions/S3-phase-<n>-tasks.md`
(for example, `S3-phase-2-tasks.md`).

The tasks must be specific enough that a contributor can start, test, and finish them without
guessing. Favor small, independently verifiable items over broad implementation buckets.

## Task checklist

List every task needed to complete the phase. Each task should be small, concrete, and testable.

- [ ] Task 1 — action verb + concrete outcome — owner: @name — validation: exact check or command
- [ ] Task 2 — action verb + concrete outcome — owner: @name — validation: exact check or command
- [ ] Task 3 — action verb + concrete outcome — owner: @name — validation: exact check or command

For each task, include:

- What will change
- What will not change
- The owner (if known)
- The verification method
- Any dependencies on earlier tasks

## Validation

Describe the tests, checks, or manual verification steps required to accept the phase as complete.
Include command examples, expected outputs, or acceptance criteria where possible.
