# S1 — Project Outline (Template)

Use this template to capture the initial alignment for a new project or major feature. This file
serves as the primary source-of-truth for high-level decisions about the project's purpose, scope,
and constraints.

- Date:

## Idea

Describe the core idea in one or two paragraphs.

## Expected behavior

Describe the observable behavior the project should deliver when the main goals are met.

## Constraints

List key constraints (technical, legal, time, resources) that bound the solution space.

## Non-goals

Explicitly list what this project will not do — helps to prevent scope creep.

## Assumptions

Record assumptions that must hold true for the design and plan to remain valid.

## Open questions

List outstanding questions that block implementation or require decisions.
